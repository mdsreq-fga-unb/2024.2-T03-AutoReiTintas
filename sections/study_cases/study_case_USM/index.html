    USM

    User Story Mapping é uma técnica usada para organizar e visualizar as funcionalidades de um produto a partir da perspectiva do usuário, com o objetivo de entender melhor como as diversas partes do sistema se conectam para entregar valor ao usuário. A ideia central é mapear histórias de usuários em um formato visual que facilita a compreensão do fluxo de trabalho, das prioridades e das interações dentro de um produto

    Board Iterativo

    Contexto

    A EduConnect busca transformar a forma como instituições educacionais gerenciam projetos interdisciplinares e atividades extracurriculares, promovendo uma integração eficiente e colaborativa entre todos os envolvidos no processo educacional. A EduConnect é uma plataforma digital que facilita a criação, gestão e avaliação de projetos educacionais interdisciplinares e atividades extracurriculares.

    Características principais:

    • Ambiente integrado para coordenação de projetos educacionais.
    • Abordagem colaborativa entre professores, alunos, pais, coordenadores e especialistas externos.
    • Promoção de projetos interdisciplinares que aumentam o engajamento e a aprendizagem dos alunos.
    • Coordenação eficiente entre gestores, professores e stakeholders.

    Visão Geral das Atividades

    Professores e Coordenadores
    • Planejamento e definição de:
      • Objetivos pedagógicos
      • Cronograma, escopo e recursos
      • Critérios de avaliação e rubricas
      • Entregas específicas e possíveis riscos
    • Desenvolvimento de materiais didáticos
    • Acompanhamento e geração de relatórios detalhados
    • Organização de apresentações finais
    Alunos
    • Participação em projetos interdisciplinares.
    • Colaboração em atividades.
    • Recebimento de feedback construtivo.
    • Participação em eventos e feiras.
    Especialistas Externos
    • Participação em reuniões de planejamento.
    • Contribuição com conhecimentos especializados.
    • Feedback sobre o impacto das atividades desenvolvidas.
    • Colaboração na criação de materiais didáticos.
    Pais/Responsáveis
    • Recebimento de informações detalhadas sobre o projeto.
    • Acompanhamento regular do progresso dos filhos.
    • Participação em eventos e reuniões.
    • Contribuição com experiências e conhecimentos.

    Critérios de aceitação ( 15 )

    • US's escolhidas: US01, US02, US05, US06, US13, US14, US15, US22, US23, US24, US25, US26, US32, US33 e US34.

    US01 - Como professor, quero criar disciplina para poder registrar quais disciplinas utilizarei no meu projeto

    • Os campos obrigatórios na criação de disciplina são:  nome, seu nome abreviado e descrição
    • O sistema deve fornecer um formulário para cadastrar uma nova disciplina.
    • O usuário deve selecionar a qual curso a disciplina pertence.
    • O sistema deve validar os dados antes de salvar, impedindo o cadastro de disciplinas com informações incompletas ou duplicadas.

    US02 - Como professor, quero editar disciplinas para corrigir algum possível erro que tenha cometido em sua criação

    • O professor pode editar os campos nome, descrição e código abreviado
    • Após a edição, um alerta deverá aparecer para confirmar a operação ou cancelá-la
    • Após a alteração, uma notificação contendo os novos valores deve ser enviada aos professores que ministram as disciplinas

    US05 - Como professor, quero criar projetos para que os alunos participem

    • Os campos necessários são nome do projeto, os professores participantes, disciplinas, data e atividades
    • Ao criar um projeto ele deverá ser notificado a todos os professores, coordenadores, alunos e especialistas externos.
    • O status inicial do projeto deve ser configurado como "Em andamento" ou outro status adequado conforme o fluxo do projeto.
    • Apenas usuários com permissão para criar projetos (ex.: professores e coordenadores) devem ter acesso a essa funcionalidade.

    US06 - Como professor, quero editar projetos para alterar dados nele caso haja alguma incoerência

    • Os campos os quais deverão ser editados são: nome, os professores participantes, disciplinas e atividades do projeto
    • O usuário deve poder acessar os detalhes de um projeto a partir da lista de projetos.
    • O sistema deve validar as edições, especialmente quanto ao formato de datas, integridade dos campos obrigatórios e não permitir datas inconsistentes
    • Somente usuários com permissões adequadas (ex.: professores e coordenadores) devem ter acesso para editar um projeto.

    US13 - Como professor, quero buscar alunos para adicioná-los no projeto

    • O sistema deve disponibilizar um campo de busca para localizar alunos.
    • A busca deve retornar apenas alunos que correspondam aos critérios inseridos.
    • O sistema deve exibir lista por padrão em ordem alfabética
    • O sistema deve filtrar lista de alunos por nome, turma, ano, matrícula e projetos
    • O sistema deve exibir perfil do aluno ao selecioná-lo
    • Se nenhum aluno for encontrado, o sistema deve exibir uma mensagem clara informando que não há resultados para a busca.

    US14 - Como professor, quero receber as solicitações dos alunos de participação do projeto para adicioná-los ao projeto

    • Receber solicitação de participação de aluno via email
    • O sistema deve permitir que os alunos enviem solicitações para participar de um projeto
    • O responsável pelo projeto deve ser notificado imediatamente quando uma nova solicitação for recebida
    • O responsável pelo projeto deve visualizar todas as solicitações pendentes em uma tela específica
    • Cada solicitação deve conter o nome do aluno, o motivo do interesse, data do envio, ano, turma e histórico de participação de projetos
    • O professor deve ter a opção de aceitar ou recusar a solicitação
    • O professor deve poder acessar um histórico de solicitações aceitas e recusadas

    US15 - Como professor, quero notificar o aluno sobre a participação do projeto para que ele saiba que foi aceito no projeto

    • O aluno deve receber uma notificação quando for adicionado a um projeto.
    • A notificação deve ser enviada por e-mail e pelo sistema em questão(site, aplicativo, etc.)
    • A mensagem deve conter o nome do projeto, a data de início e o responsável pelo convite
    • O sistema deve permitir que o aluno visualize a notificação e marque como lida.
    • O aluno deve poder acessar notificações anteriores em um histórico dentro do sistema.
    • Caso o aluno esteja logado no sistema, a notificação deve ser exibida em tempo real.

    US22 - Como aluno, quero solicitar a entrada no projeto para que eu participe dele

    • o aluno deve informar seu nome, matrícula, ano, turma e motivação para realizar a solicitação
    • Após enviar a solicitação, a cópia dessa solicitação poderá ser enviada ao aluno solicitante
    • Após enviar​ a solicitação, ela deverá notificar o professor responsável pelo projeto

    US23 - Como aluno, quero confirmar participação no projeto para que possa efetivamente participar no projeto

    • O aluno deve receber uma notificação informando que foi convidado para participar do projeto.
    • O aluno deve ter a opção de aceitar ou recusar o convite diretamente na notificação ou em uma tela específica.
    • O convite deve ter um prazo máximo para resposta, após o qual a participação é automaticamente cancelada.
    • Após aceitar o convite, o aluno deve receber uma confirmação de participação no projeto.
    • O nome do aluno deve ser adicionado à lista de participantes do projeto assim que ele confirmar a participação.
    • O responsável pelo projeto deve ser notificado quando o aluno confirmar ou recusar a participação.
    • Após confirmar a participação, o aluno deve ter acesso aos detalhes do projeto, como descrição, tarefas e membros.

    US24 - Como aluno, quero realizar as atividades para que o projeto progrida

    • O aluno deve visualizar uma lista de atividades do projeto em formato de checklist.
    • O aluno deve poder marcar uma atividade como concluída ao finalizá-la.
    • Se necessário, o aluno deve poder desmarcar uma atividade concluída para correção ou retrabalho.
    • O sistema deve exibir o percentual de atividades concluídas para acompanhar o progresso do projeto.
    • O responsável pelo projeto deve ser notificado quando uma atividade for concluída.
    • Apenas usuários autorizados devem poder adicionar, editar ou excluir atividades do checklist.
    • Cada atividade pode ter um prazo de conclusão, e o sistema deve alertar sobre atividades atrasadas.

    US25 - Como aluno, quero criar atividades nos projetos para definir o passo a passo de execução do projeto

    • São obrigatórios os campos nome, assunto, período de execução, status e participantes
    • Ao criar a atividade no projeto, todos os seus integrantes deverão ser notificados sobre essa criação
    • Os campos deverão ser validados antes de serem cadastrados
    • A partir do momento em que forem criados os campos, eles deverão possuir o status de 'a fazer'

    US26 - Como aluno, quero editar os campos das atividades dos projetos para corrigir um eventual erro

    • Os campos que poderão ser editados são: nome, assunto, período de execução, status, e participantes
    • Ao editar a atividade no projeto, todos os seus integrantes deverão ser notificados sobre essa edição

    US32 - Como aluno, quero encaminhar feedback sobre o projeto para que encontrem soluções para as questões que observei

    • ​Os campos que devem ser preenchidos do feedback do aluno são: Qualidade de aprendizado, qualidade das aulas, qualidade dos materiais de apoio, relação com os membros do projeto, feedback individual de cada membro todos variando de 0 (baixo) para 10 (alto).
    • O aluno deve ser capaz de escrever melhorias sobre algum aspecto do projeto
    • O aluno deve ser capaz de armazenar as próprias respostas.
    • O feedback deve ter salvamento automático de respostas do campo

    US33 - Como coordenador pedagógico, quero gerar relatórios sobre o andamento do projeto para que eu possa analisar e sugerir melhorias à didática dos professores e especialistas

    • O relatório deve ser gerado em um formato claro e estruturado, como PDF ou outro formato adequado, para facilitar a análise e o compartilhamento.
    • O relatório deve incluir métricas como:

      Progresso atual das atividades do projeto.
      Desempenho individual dos professores e especialistas envolvidos.
      Identificação de atividades concluídas, em andamento e pendentes.
      Feedbacks recebidos dos alunos relacionados ao projeto.

    • O sistema deve permitir a definição do período de tempo (datas de início e fim) para o qual o relatório será gerado.
    • Deve ser possível selecionar um ou mais projetos específicos para os quais o relatório será gerado.
    • O coordenador pedagógico deve conseguir acessar a funcionalidade de geração de relatórios a partir do painel principal do sistema.

    US34 - Como coordenador pedagógico, quero visualizar o progresso das atividades dos projetos para que eu possa analisar se estão sendo desenvolvidas conforme o planejado

    • ​o progresso dos projetos deve ser mostrado via porcentagem
    • O progresso precisa exibir quais atividades do projeto foram concluídas
    • A visualização deve comparar as atividades que se esperava ter concluídas na data presente com as atividades atualmente concluídas
    • O progresso deve mostrar as mudanças que ocorreram na equipe como saída entradas de alunos, professores e especialistas
